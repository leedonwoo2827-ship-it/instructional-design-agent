# 디자인 슬라이드 이미지 소스 (Openverse · Unsplash)

STEP 4 ②(이미지·레이아웃 정리)에서 슬라이드 사진을 자동 수집한다.
구현: [core/image_search.py](../core/image_search.py). 키 설정: 사이드바 › 연결 설정.

## 동작 규칙
- **Unsplash Access Key 가 있으면 Unsplash 우선**, 없거나 실패하면 **Openverse 폴백**.
- 키는 사이드바에 입력 → `data/user_settings.json`(로컬, `.gitignore`)에 저장. 넣고 빼기 자유.
- 환경변수 `UNSPLASH_ACCESS_KEY` 로도 설정 가능.
- 사진 실패/레이트리밋이면 그 슬라이드만 사진 생략(덱은 완성). 슬라이드 수는 항상 보존.

## Openverse (기본 · 키 불필요)
- **비용 0**, 키 불필요. CC/퍼블릭 도메인 이미지.
- 익명 요청은 레이트리밋이 있어 대량 생성 시 일부 누락 가능.
- 품질은 편차 큼(“30%만 만족” 피드백).

## Unsplash (선택 · 품질↑)
- **비용: 무료 ($0).** 그림당·회당·월정액 과금 없음. Unsplash License(상업적 사용 포함 무료).
- **제약 = 시간당 요청 수(rate limit):**
  - Demo(가입 즉시): **시간당 50 요청**
  - Production(앱 심사·승인 후, 무료): **시간당 5,000 요청**
- 덱 1개 ≈ 사진 슬라이드 ~25장 = 검색 ~25요청. Demo면 1시간 1~2덱 수준.
- **의무(앱이 자동 처리):**
  1. Access Key 발급 — https://unsplash.com/developers (무료 앱 등록)
  2. 저작자 표시 — `Photo by [작가] on Unsplash` + 작가·사진 링크(UTM) → 이미지 출처(.txt)에 자동 기재
  3. 사용 시 download 트래킹 엔드포인트 1회 호출 → 자동
  - 참고: 옛 `source.unsplash.com` 무키 엔드포인트는 2023년 폐지 → 키 기반 API 필수.
- **한계:** Unsplash/Openverse 모두 일반 스톡 사진. 논문 도판·학술 표 같은 건 무료 소스로 자동 확보 불가(표가 필요한 내용은 `table` 타입으로 실제 표 생성).

## 키 발급·적용 순서 (Unsplash)
1. https://unsplash.com/developers → New Application → **Access Key** 복사
2. 앱 사이드바 › 연결 설정 › **Unsplash Access Key** 붙여넣기 → 저장
3. STEP 4 ② 재실행 → 사진이 Unsplash에서 수집, 출처 .txt에 표기

## 향후
- 관련도·품질을 더 올리려면 유료 스톡 API(예: Pexels 무료, Shutterstock 유료) 추가 폴백도 가능.
- 이미지 검색어(image_query)는 아트디렉터(LLM)가 이론 특화 영어로 생성 — [core/deck_builder.py](../core/deck_builder.py) `_ART_RULES` 참조.
